  <h1 class="cell-title">MNIST with PyTorch</h1>

  <p>This notebook trains a <strong>neural network</strong> to read handwritten digits — the numbers 0 through 9. You feed it a picture of someone's handwriting, and it tells you which digit it is.</p>

  <p>By the time you reach the last cell, you'll have a model that reads handwriting correctly <strong>~97% of the time</strong>. And you'll understand exactly how it learned to do that.</p>

  <div class="callout concept">
    <span class="callout-label">🧠 The Big Picture</span>
    <p class="callout-body">A neural network is just a <strong>function full of numbers</strong>. It starts out dumb — completely random. Training is the process of showing it thousands of examples and slowly adjusting those numbers until the function gets the right answer. That's it. Everything else is details.</p>
  </div>

  <h1 class="cell-title">The Training Loop</h1>

  <p>This is the most important section. <strong>The training loop is how the network learns.</strong> Every neural network ever trained — from MNIST classifiers to GPT — runs inside some version of this five-step loop.</p>

  <div class="loop-diagram">
    <div class="loop-node n1"><div class="loop-num">step 1</div><div class="loop-name">Zero Grad</div><div class="loop-sub">Clear old gradient info</div></div>
    <div class="loop-arrow-h">→</div>
    <div class="loop-node n2"><div class="loop-num">step 2</div><div class="loop-name">Forward</div><div class="loop-sub">Run data through network</div></div>
    <div class="loop-arrow-h">→</div>
    <div class="loop-node n3"><div class="loop-num">step 3</div><div class="loop-name">Loss</div><div class="loop-sub">Measure how wrong we are</div></div>
    <div class="loop-arrow-h">→</div>
    <div class="loop-node n4"><div class="loop-num">step 4</div><div class="loop-name">Backward</div><div class="loop-sub">Trace what caused the error</div></div>
    <div class="loop-arrow-h">→</div>
    <div class="loop-node n5"><div class="loop-num">step 5</div><div class="loop-name">Step</div><div class="loop-sub">Nudge parameters to improve</div></div>
  </div>

  <div class="five-steps">
    <div class="fs-card">
      <div class="fs-num">1</div>
      <div class="fs-body">
        <code class="fs-code">optimizer.zero_grad()</code>
        <div class="fs-explain">Clears any gradient values left over from the previous batch. PyTorch accumulates gradients by default — if you skip this, old gradients pile up across batches and corrupt training completely.</div>
      </div>
      <div class="fs-why">most forgotten step</div>
    </div>
    <div class="fs-card">
      <div class="fs-num">2</div>
      <div class="fs-body">
        <code class="fs-code">outputs = model(images)</code>
        <div class="fs-explain">Runs the 64 images through all the layers. Returns a [64, 10] tensor — 64 rows, each with 10 logit scores. PyTorch secretly records every operation so it can reverse them later.</div>
      </div>
      <div class="fs-why">forward pass</div>
    </div>
    <div class="fs-card">
      <div class="fs-num">3</div>
      <div class="fs-body">
        <code class="fs-code">loss = criterion(outputs, labels)</code>
        <div class="fs-explain">Compares the network's 10 scores against the true labels. Produces one number: how wrong was the network on average across these 64 images? Higher = worse. We want this to fall.</div>
      </div>
      <div class="fs-why">cross-entropy loss</div>
    </div>
    <div class="fs-card">
      <div class="fs-num">4</div>
      <div class="fs-body">
        <code class="fs-code">loss.backward()</code>
        <div class="fs-explain">Works backwards through the recorded operations using calculus (chain rule). Computes a gradient for every one of the ~270,000 parameters — a number saying "if you increase this parameter, does the loss go up or down, and by how much?"</div>
      </div>
      <div class="fs-why">backpropagation</div>
    </div>
    <div class="fs-card">
      <div class="fs-num">5</div>
      <div class="fs-body">
        <code class="fs-code">optimizer.step()</code>
        <div class="fs-explain">Uses the gradients to nudge every parameter a tiny bit in the direction that reduces the loss. Adam (our optimizer) does this smartly — it uses a different step size for each parameter based on its history.</div>
      </div>
      <div class="fs-why">gradient descent</div>
    </div>
  </div>

  <div class="code-block">
    <div class="code-toolbar">
      <div class="tb-dot r"></div><div class="tb-dot y"></div><div class="tb-dot g"></div>

<span class="sy-kw">def</span> <span class="sy-fn">train_one_epoch</span>(model, loader):
    model.<span class="sy-fn">train</span>()  <span class="sy-cm"># puts network in training mode (important for Dropout/BatchNorm)</span>
    total_loss, correct, total = <span class="sy-nm">0</span>, <span class="sy-nm">0</span>, <span class="sy-nm">0</span>

    <span class="sy-kw">for</span> images, labels <span class="sy-kw">in</span> loader:  <span class="sy-cm"># each iteration = one batch of 64 images</span>
        images, labels = images.<span class="sy-fn">to</span>(device), labels.<span class="sy-fn">to</span>(device)

        optimizer.<span class="sy-fn">zero_grad</span>()               <span class="sy-cm"># ① clear old gradients</span>
        outputs = model(images)             <span class="sy-cm"># ② forward pass → [64, 10]</span>
        loss    = criterion(outputs, labels) <span class="sy-cm"># ③ compute loss scalar</span>
        loss.<span class="sy-fn">backward</span>()                     <span class="sy-cm"># ④ compute all gradients</span>
        optimizer.<span class="sy-fn">step</span>()                    <span class="sy-cm"># ⑤ update all parameters</span>

        total_loss += loss.<span class="sy-fn">item</span>()          <span class="sy-cm"># .item() extracts the plain number</span>
        preds  = outputs.<span class="sy-fn">argmax</span>(dim=<span class="sy-nm">1</span>)    <span class="sy-cm"># pick the highest-scoring class per image</span>
        correct += (preds == labels).<span class="sy-fn">sum</span>().<span class="sy-fn">item</span>()
        total   += labels.<span class="sy-fn">size</span>(<span class="sy-nm">0</span>)

    <span class="sy-kw">return</span> total_loss / <span class="sy-bi">len</span>(loader), correct / total</pre>
  </div>

  <div class="callout warn">
    <span class="callout-label">⚠️ The #1 beginner mistake: forgetting zero_grad()</span>
    <p class="callout-body">PyTorch accumulates (adds) gradients every time you call <code>loss.backward()</code>. This is a feature for advanced use cases, but for a basic training loop it's a trap. If you forget <code>zero_grad()</code>, each batch's gradients get added to all previous batches' gradients. After 100 batches, every parameter's gradient is 100× too large, and training explodes. Always call it.</p>
  </div>

  <div class="callout tip">
    <span class="callout-label">💡 What is the learning rate (lr=1e-3)?</span>
    <p class="callout-body">The learning rate controls how big each parameter update is. <code>1e-3</code> means 0.001. Think of it like step size when walking down a hill: too large and you overshoot the bottom and go back up; too small and it takes forever. 0.001 is a safe, well-tested default for Adam on most tasks.</p>
  </div>
